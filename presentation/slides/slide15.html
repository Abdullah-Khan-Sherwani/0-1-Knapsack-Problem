Conclusion — Match the Algorithm to the Constraint

Tiny n (≤ 25)

Brute force — sanity check / oracle.

Need item set

Tabulation — fastest exact + recoverable.

Memory-bound

Space-Optimised — O(W) only.

Huge W, speed critical

Greedy — independent of W, ≥ OPT/2.

Need provable bound

FPTAS — pick ε, get (1−ε)·OPT.

Default choice

Tabulation if W small, FPTAS if W huge.

The real lesson

"Best" depends on the instance. Empirical structure (correlation, W magnitude, n) determines the winning algorithm — there is no universal answer, only well-justified trade-offs.
