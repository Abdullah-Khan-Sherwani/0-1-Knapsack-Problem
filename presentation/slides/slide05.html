1. Brute Force (Pure Recursion)

How it works

For each item: take it or skip it. Recurse on the remaining n−1 items, return the max of the two branches. No caching — every subproblem is re-solved from scratch.

Time: O(2ⁿ)   |   Space: O(n) recursion stack

Pros

  • Trivial correctness; useful as a reference for testing other algorithms.
  • No data structures, no memory pressure.

Cons

  • Explodes after n ≈ 25 (2²⁵ ≈ 33 million calls).
  • Same subproblem solved many times.
