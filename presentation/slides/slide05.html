2. Tabulation

How it works

Build a 2-D table tab[i][w] iteratively from i = 0…n and w = 0…W. Each cell is the optimum using the first i items with capacity w. No recursion, no stack overhead.

Time: O(n · W)   |   Space: O(n · W)

Pros

  • Iterative — fastest in practice among the exact DPs (no call overhead).
  • The full table lets us backtrack and reconstruct the chosen items.

Cons

  • Same O(nW) memory as memoization.
  • Computes every cell, even ones the recursion would have skipped.

Core Logic

for i in range(1, n + 1):
  for w in range(1, capacity + 1):
    if weights[i-1] <= w:
      include = (values[i-1]
                 + tab[i-1][w - weights[i-1]])
      exclude = tab[i-1][w]
      tab[i][w] = max(include, exclude)
    else:
      tab[i][w] = tab[i-1][w]
