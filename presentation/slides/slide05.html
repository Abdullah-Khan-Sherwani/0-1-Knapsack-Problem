2. Tabulation

How it works

Build a 2-D table tab[i][w] iteratively from i = 0…n and w = 0…W. Each cell is the optimum using the first i items with capacity w. No recursion, no stack overhead.

Time: O(n · W)   |   Space: O(n · W)

Over Brute Force

  • Reduces O(2ⁿ) to O(n·W) — handles n = 500 where BF collapses at n = 25.
  • Each subproblem solved exactly once; BF recomputes overlapping subproblems repeatedly.

Over Memoization

  • No recursion stack — never hits Python's depth limit; no setrecursionlimit needed.
  • No per-call frame allocation overhead.

Core Logic

for i in range(1, n + 1):
  for w in range(1, capacity + 1):
    if weights[i-1] <= w:
      include = (values[i-1]
               + tab[i-1][w - weights[i-1]])
      exclude = tab[i-1][w]
      tab[i][w] = max(include, exclude)
    else:
      tab[i][w] = tab[i-1][w]

Empirical discrepancy: Textbooks claim tabulation is fastest in practice due to cache locality — true in C++. In our Python benchmarks, memoization was faster across every category and n value (1.02× to 45× slower). Memoization skips unreachable (i, w) states; tabulation always fills the full n×W table.
