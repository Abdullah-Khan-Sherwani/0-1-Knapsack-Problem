3. FPTAS — Fully Polynomial-Time Approximation Scheme

How it works

Pick ε ∈ (0, 1). Scale all values by K = εP/n where P = max value. Round down to integers, then run a profit-indexed 2-D DP: dp[i][p] = min weight to achieve scaled profit p using first i items. Backtrack to recover chosen items.

Time: O(n³ / ε)   |   Space: O(n³ / ε)   |   Guarantee: ≥ (1−ε) · OPT

Pros

  • Tunable — pick any ε, get a guarantee proportional to it.
  • Polynomial in n and 1/ε — beats DP when W is huge but n is small.

Cons

  • Approximation only — never guaranteed to return exact OPT.
  • Slower than greedy; O(n³/ε) space from the 2-D backtracking table.

Core Logic

K = (eps * max(values)) / n
scaled = [floor(v/K) for v in values]

for i in range(1, n+1):
  vi = scaled[i-1]
  for p in range(max_profit+1):
    dp[i][p] = dp[i-1][p]
    if p >= vi:
      dp[i][p] = min(dp[i][p],
        dp[i-1][p-vi] + weights[i-1])
