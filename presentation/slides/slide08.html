5. Greedy Approximation

How it works

Sort items by value/weight ratio descending. Pack greedily until the next one doesn't fit. To guarantee ≥ OPT/2, return max(greedy_pack, best_single_item).

Time: O(n log n)   |   Space: O(n)   |   Guarantee: ≥ OPT / 2

Pros

  • Lightning fast — no DP table at all; independent of W.
  • Trivial to implement; uses only sorting.

Cons

  • Only a 50% guarantee in the worst case.
  • Performs poorly on strongly-correlated instances (all ratios similar).

Core Logic

order = sorted(range(n),
  key=lambda i:
    values[i]/weights[i],
  reverse=True)

total_w = total_v = 0
for i in order:
  if total_w + weights[i] <= W:
    total_w += weights[i]
    total_v += values[i]

return max(total_v, max(values))
