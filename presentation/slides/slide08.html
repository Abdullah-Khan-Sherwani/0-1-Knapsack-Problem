4. Space-Optimised DP

How it works

Tabulation only ever reads row i−1 to compute row i, so we keep just one 1-D array dp[w]. Iterate w from W down to weights[i] so we don't overwrite values still needed.

Time: O(n · W)   |   Space: O(W)

Pros

  • Same speed as tabulation but huge memory win — only one row.
  • Best choice when W is large but per-instance memory matters.

Cons

  • Cannot recover the item set from a single row — that information is lost.
  • Still pseudo-polynomial in W.

Why "value only"?

The 1-D rolling array discards the history needed to reconstruct which items were chosen.

To recover items you'd need a parent table — defeating the space win.
