Benchmark Methodology & Test Cases

Environment

  • Python 3, standard library only
  • Single machine, time.perf_counter()
  • Median of multiple instances per (n, ratio) cell

Test Suite — Pisinger kplib

  • 14 instance categories (correlation structures)
  • n ∈ {5, 10, 25, 50, 100, 200, 500, …}
  • Two value ranges: R01000 and R10000
  • Capacity = sum(weights) / 2

Why these categories?

  • Uncorrelated — easiest, ratios spread out
  • Strongly correlated — hardest, ratios all similar
  • Subset Sum — values = weights, special case
  • Spanner / Profit Ceiling / Circle — pathological inputs

Skip Thresholds (avoid OOM)

  • BF skipped for n > 25 (2ⁿ explodes)
  • DP variants skipped at n·W > 15M–30M
  • Greedy never skipped

Cross-check: all four exact algorithms must produce the same value on every instance — flagged automatically if they disagree.
