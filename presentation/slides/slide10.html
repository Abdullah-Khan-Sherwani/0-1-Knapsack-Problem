6. FPTAS — Fully Polynomial-Time Approximation Scheme

How it works

Pick ε ∈ (0, 1). Scale all values by K = εP/n where P = max value. Round to integers, then run a profit-indexed DP: dp[p] = min weight to achieve scaled profit p. Largest feasible p gives the answer.

Time: O(n³ / ε)   |   Space: O(n² / ε)   |   Guarantee: ≥ (1−ε) · OPT

Pros

  • Tunable — pick any ε, get a guarantee proportional to it.
  • Polynomial in n and 1/ε — beats DP when W is huge but values are small.

Cons

  • Slower than greedy; harder to implement than DP.
  • Like SpaceOpt, item set hard to recover from a 1-D profit array.
