Results: Runtime vs Problem Size

Data shown: benchmark_n100.csv · both ratios · n = 10–100 · 13Synthetic excluded

What the chart shows

  • BruteForce spikes off the chart at n = 25 — the O(2ⁿ) wall.
  • Memoization consistently above tabulation — recursion overhead and cache misses add up.
  • Greedy is essentially flat — O(n log n), completely W-independent.
  • FPTAS stays well below DP — scaled problem size keeps its DP table small.

Discrepancy worth noting

As n grows large, tabulation tends to outperform memoization in practice because it avoids recursion overhead and has better cache locality.
