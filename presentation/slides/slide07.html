4. Memoization

How it works

Same recursion as brute force, plus a memo[n+1][W+1] table that caches each (i, w) subproblem. Each state is solved once → total work proportional to table size.

Time: O(n · W)   |   Space: O(n · W)

Pros

  • Skips unreachable (i, w) states — only solves subproblems that are actually needed.
  • Faster than tabulation when W is large relative to n: skips most of the table, doing far less work.

Cons

  • Recursion call overhead per subproblem — at large n, per-frame cost accumulates and tabulation's loops win.
  • Recursion depth bounded by n — deep stacks require runtime configuration.

Core Logic

def knapsack_memoization(cap, n, values,
                          weights, memo):
  if memo[n][cap] is not None:
    return memo[n][cap]
  if n == 0 or cap == 0:
    result = (0, [])
  elif weights[n-1] > cap:
    result = knapsack_memoization(
                 cap, n-1, values, weights, memo)
  else:
    inc, _ = knapsack_memoization(
                 cap-weights[n-1], n-1,
                 values, weights, memo)
    inc += values[n-1]
    exc, _ = knapsack_memoization(
                 cap, n-1, values, weights, memo)
    result = (inc, ...) if inc >= exc else (exc, ...)
  memo[n][cap] = result
  return result
