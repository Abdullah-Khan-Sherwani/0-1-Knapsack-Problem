4. Memoization

How it works

Same recursion as brute force, plus a memo[n+1][W+1] table that caches each (i, w) subproblem. Each state is solved once → total work proportional to table size.

Time: O(n · W)   |   Space: O(n · W)

Pros

  • Reuses brute-force logic — easy to write once you have the recursion.
  • Massive speed-up: from O(2ⁿ) to O(nW).

Cons

  • Pseudo-polynomial — runtime depends on the value of W, not its bit-length.
  • Recursion depth grows with n (Python: needs setrecursionlimit).

Core Logic

def solve(i, w):
  if i == 0 or w == 0:
    return 0
  if memo[i][w] is not None:
    return memo[i][w]
  if weights[i-1] > w:
    res = solve(i-1, w)
  else:
    res = max(solve(i-1, w),
      values[i-1] +
        solve(i-1, w-weights[i-1]))
  memo[i][w] = res
  return res
