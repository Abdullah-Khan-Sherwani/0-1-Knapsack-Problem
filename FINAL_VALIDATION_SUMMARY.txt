6. CRITICAL BUG FOUND AND FIXED ✗→✓
═══════════════════════════════════════════════════════════════════════════════

BUG: parse_kp.py had columns reversed

BEFORE (WRONG):
  Line 8-9: weights.append(int(parts[0]))      ← Read VALUE as weight
            values.append(int(parts[1]))        ← Read WEIGHT as value

AFTER (FIXED):
  Line 8-9: values.append(int(parts[0]))       ← Correctly read VALUE
            weights.append(int(parts[1]))      ← Correctly read WEIGHT

kplib Format (per README):
  ```
  n
  c
  p_1 w_1    (p = price/value, w = weight)
  ...
  ```

Impact of Fix:
  ✗ BEFORE: Benchmark results would have been completely wrong
  ✓ AFTER: Benchmarks will now run with correct data

VERIFICATION:
  Sample .kp file entry: "845 804"
  BEFORE parsing: value=804, weight=845 (WRONG!)
  AFTER parsing:  value=845, weight=804 (CORRECT!)
