TEST 3: test_brute_force_forced.py
════════════════════════════════════════════════════════════════════════════════

Purpose:
  Force brute force execution to demonstrate it works correctly, despite
  being exponential. Shows why DP is needed for larger problems.

What It Tests:
  Brute force on single small instance and compares to Space-Optimised DP

Problem:
  Single kplib instance: n=50, capacity=14,778
  This triggers 2^50 = 1,125,899,906,842,624 operations

Warning:
  This test will take a VERY LONG TIME on n=50.
  Recommended: Only run for demonstration if you have patience/time.

How to Run:
  python testing/test_brute_force_forced.py

Expected Output:
  ✓ Both algorithms return same result
  ✗ Brute Force will be MUCH SLOWER than DP

Runtime: Depends on system, potentially hours for n=50

What It Validates:
  ✓ Brute force algorithm is correct
  ✓ Demonstrates exponential complexity practically
  ✓ Justifies DP approach for larger problems

Note: For presentations, mention this test exists but don't actually run it
      on large n. It's mainly for theoretical validation.

════════════════════════════════════════════════════════════════════════════════
INTERPRETING RESULTS
════════════════════════════════════════════════════════════════════════════════

What You Want to See:

✓ All exact algorithms (BF, MEMO, TAB, SO) return IDENTICAL values
  If different: Algorithm bug exists
  If identical: All algorithms correct

✓ Greedy results are LOWER than or EQUAL to exact algorithms
  If greedy > exact: Algorithm bug
  If greedy ≤ exact: Greedy working correctly

✓ Gap calculations (Exact - Greedy) / Exact * 100 are non-negative
  If negative: Error in gap calculation
  If non-negative: Expected behavior

✓ Greedy gap varies by problem type
  Uncorrelated: Should be < 1%
  Weakly Correlated: Should be < 1%
  Strongly Correlated: Can be 0-5%

✓ Runtimes follow complexity patterns
  Brute Force: Exponential (slow on larger n)
  DP: O(n*W) (medium speed)
  Space-Optimised: O(n*W) (faster than DP)
  Greedy: O(n log n) (very fast)

════════════════════════════════════════════════════════════════════════════════
TROUBLESHOOTING
════════════════════════════════════════════════════════════════════════════════

If test fails:

1. Algorithms return different values
   Issue: Algorithm bug or data inconsistency
   Solution: Check parse_kp.py is using correct column order

2. Greedy > Exact algorithm
   Issue: Greedy algorithm bug
   Solution: Verify greedy ratio calculation

3. Timeout on large instances
   Issue: Memory exceeded or computation too long
   Solution: Expected behavior, tests auto-skip when needed

4. Import errors
   Issue: Missing module
   Solution: Ensure src/ folder in Python path (tests add it automatically)

5. File not found errors
   Issue: testcases/ folder not in expected location
   Solution: Run tests from project root directory
