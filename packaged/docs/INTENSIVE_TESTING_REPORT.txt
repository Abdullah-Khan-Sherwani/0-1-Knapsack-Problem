STANDARD PROCEDURE COMPLIANCE
═══════════════════════════════════════════════════════════════════════════════

Each algorithm was verified to follow standard DP/algorithmic approaches:

✓ BRUTE FORCE
  - Standard recursive approach with 0/1 choice at each item
  - Base case: (n=0 or W=0) → 0
  - Recursive case: max(exclude, include_if_fits)
  - Time: O(2^n), Space: O(n)

✓ MEMOIZATION
  - Standard top-down DP pattern
  - Initializes memo table to None
  - Caches results before returning
  - Recursive calls with reduced problem size
  - Time: O(n×W), Space: O(n×W)

✓ TABULATION
  - Standard bottom-up DP pattern
  - Builds table iteratively from base cases
  - Backtracks to find selected items
  - Returns both value and item selection
  - Time: O(n×W), Space: O(n×W)

✓ SPACE-OPTIMISED
  - Standard 1D array optimization of tabulation
  - Uses single array instead of 2D table
  - Iterates capacity in reverse to maintain correctness
  - Time: O(n×W), Space: O(W)

✓ GREEDY
  - Standard greedy approach for approximation
  - Sorts by value/weight ratio (descending)
  - Greedily selects items that fit
  - Time: O(n log n), Space: O(n)

PROBLEM CHARACTERISTICS IMPACT
═══════════════════════════════════════════════════════════════════════════════

How Problem Type Affects Algorithms:

UNCORRELATED PROBLEMS
- Values and weights vary independently
- Greedy approximation: Excellent (0-0.37% gap)
- Why: No systematic pattern for greedy to exploit wrongly
- Best Algorithm: Space-Optimised DP for speed or Greedy for ultra-fast approx

WEAKLY CORRELATED
- Slight positive correlation (higher value ↔ higher weight typically)
- Greedy approximation: Very Good (0-0.36% gap)
- Why: Light correlation helps greedy make reasonable choices
- Best Algorithm: Space-Optimised DP

STRONGLY CORRELATED
- High correlation (value ≈ weight proportionally)
- Greedy approximation: Good but Weaker (0-4% gap)
- Why: Greedy gets fooled by value/weight ratios that don't reflect utility
- Example: 2 items with same ratio but different absolute values
- Best Algorithm: Must use Tabulation/Space-Optimised for optimality
